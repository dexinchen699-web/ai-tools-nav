    NemoClaw 保姆级入门教程：英伟达开源 AI Agent 平台，零代码部署你的第一个企业智能体

    读完本文，你能在本地或云端部署一个基于 NemoClaw 的 AI Agent，连接工具、执行任务，全程不写一行代码。前置条件：有 Docker 环境，或者一台能上网的电脑。

    NemoClaw 是什么：英伟达为什么要做这个

    2026 年 3 月 15 日，英伟达 CEO 黄仁勋在 GTC 大会上发布了 NemoClaw。他在台上说了一句话，被很多人截图转发：

    "每家公司今天都需要有一个 OpenClaw 战略、一个 Agent 系统战略。这是新的计算机。"

    NemoClaw 是英伟达基于开源 AI Agent 框架 OpenClaw 构建的企业级平台。简单说，OpenClaw 是一个让 AI 能"自主干活"的框架（类似给 AI 装上手脚），而 NemoClaw 是英伟达在它上面加了一层企业级的安全隔离和管理能力。

    核心架构是这样的：

    graph LR A[你的指令] --> B[NemoClaw Agent] B --> C[OpenShell 安全沙箱] C --> D[工具调用：搜索/代码/文件] C --> E[模型推理：Nemotron] D --> F[执行结果] E --> F F --> G[返回给你]

    OpenShell 是 NemoClaw 的核心安全机制——每个 Agent 运行在独立的沙箱里，网络访问、文件操作、模型调用都受 YAML 策略文件控制。这是它和普通 AI 聊天工具最大的区别：你可以精确控制 Agent 能做什么、不能做什么。

    开始之前：你需要准备什么

    NemoClaw 有两种使用方式：

    • 云端托管版（nemoclaw.run）：注册账号即用，无需本地环境，适合快速体验
    • 本地部署版：需要 Docker，适合企业内网或数据不出门的场景

    本教程以云端托管版为主，本地部署版在最后单独说明。

    你需要：一个邮箱（注册账号用），仅此而已。

    Step 1：注册 NemoClaw 账号

    打开 nemoclaw.run，点击右上角"Get Started"，用邮箱注册。

    注册完成后，你会进入一个控制台界面，左侧是 Agent 列表，右侧是配置区域。

    ℹ️
    此时你应该看到：一个空的 Agent 列表，顶部有"Create Agent"按钮。

    Step 2：创建你的第一个 Agent

    点击"Create Agent"，填写以下信息：

    • Agent Name：随便起，比如"我的第一个助手"
    • Base Model：选 nemotron-nano（免费额度最大，适合入门）
    • Template：选"General Assistant"（通用助手模板）

    点击"Create"，等待约 10 秒，Agent 创建完成。

    ℹ️
    此时你应该看到：Agent 卡片出现在列表里，状态显示"Ready"。

    Step 3：给 Agent 配置工具

    点击刚创建的 Agent，进入详情页，找到"Tools"标签。

    NemoClaw 内置了几个常用工具，直接开关切换即可：

    • Web Search：让 Agent 能搜索网络（建议开启）
    • Code Executor：让 Agent 能运行代码（按需开启）
    • File Access：让 Agent 能读写文件（按需开启）

    先只开启"Web Search"，其他保持关闭。

    💡
    小技巧工具越少，Agent 的行为越可预测。入门阶段建议只开一两个工具，熟悉之后再逐步扩展。

    Step 4：测试你的 Agent

    点击"Chat"标签，进入对话界面。输入一个需要搜索的问题，比如：

    "今天 AI 领域有什么重要新闻？帮我整理成 3 条要点。"

    Agent 会自动调用 Web Search 工具，搜索最新信息，然后整理成你要求的格式返回。

    ℹ️
    此时你应该看到：Agent 先显示"Searching..."，然后返回一个结构化的 3 条要点列表。如果只返回了文字没有搜索，检查 Web Search 工具是否已开启。

    Step 5：设置安全策略（可选但推荐）

    这是 NemoClaw 区别于普通 AI 工具的核心功能。点击"Security"标签，你会看到一个 YAML 编辑器。

    默认策略已经比较安全，但你可以根据需要调整。比如，限制 Agent 只能访问特定域名：

    network:
  allowed_domains:
    - "*.google.com"
    - "*.wikipedia.org"
  block_all_others: true

    保存后，Agent 的网络访问就被限制在这两个域名范围内了。

    ⚠️
    注意修改安全策略后需要重启 Agent（点击"Restart"按钮）才能生效。

    本地部署版：3 条命令搞定

    如果你需要数据不出本地，可以用 Docker 部署：

    # 1. 拉取镜像
docker pull nvcr.io/nvidia/nemoclaw:latest

# 2. 启动服务
docker run -d -p 8080:8080 --name nemoclaw nvcr.io/nvidia/nemoclaw:latest

# 3. 打开控制台
# 浏览器访问 http://localhost:8080

    本地版和云端版功能完全一致，区别是模型推理在你自己的机器上跑，数据不会上传到英伟达服务器。

    ⚠️
    注意本地版需要至少 8GB 内存，推荐 16GB 以上。如果有 NVIDIA GPU，推理速度会快很多。

    常见问题 FAQ

    Q：NemoClaw 免费吗？
    A：云端版有免费额度（每月 100 次 Agent 调用），超出后按量计费。本地部署版完全免费，但需要自己的硬件。

    Q：NemoClaw 和 Dify、Coze 有什么区别？
    A：Dify 和 Coze 更偏向"搭建 AI 应用"，NemoClaw 更偏向"部署自主执行任务的 Agent"。前者适合做聊天机器人、知识库问答；后者适合做能自主完成多步骤任务的智能体。如果你想了解 Dify 的用法，可以看这篇教程。

    Q：OpenClaw 和 NemoClaw 是什么关系？
    A：OpenClaw 是开源的 AI Agent 框架（类似 Linux 内核），NemoClaw 是英伟达基于它构建的企业发行版（类似 Ubuntu）。你可以直接用 OpenClaw，也可以用 NemoClaw 获得更好的安全管理和技术支持。

    下一步：进阶用法

    掌握基础之后，可以尝试：

    • 连接自定义工具（通过 ToolRegistry API 注册你自己的函数）
    • 搭建多 Agent 协作流程（一个 Agent 拆任务，多个 Agent 并行执行）
    • 接入企业内部系统（数据库、内部 API、文件服务器）

    如果你对多 Agent 协作感兴趣，可以先看看这篇关于 MAS 多智能体系统的深度解析，理解背后的架构逻辑会让你上手更快。
